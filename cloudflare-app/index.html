How your key & data are handled

Written plainly, because you should not have to guess.

Your data lives on this device. Profiles, charts, conversations, journal entries and portraits are stored only in this browser. There are no accounts and no database. If you clear this browser's data, it is gone — use Backup → Export to keep a copy.

When you send a message, your message and your chart context are sent over HTTPS to this app's server (/api/chat), which forwards them to Anthropic (Claude) to write the reply, then streams it back. The server keeps no copies, has no database, and logs no conversation content. Anthropic processes API messages to generate the reply and does not use them to train its models.

If you use an access code, replies run on the app owner's API key. The owner can see total usage and cost on their key, but has no record of what you write — conversations exist only on your device.

If you use your own API key, it is stored only in this browser and sent with each request to the server, which passes it to the AI provider and does not store it. Sensible precautions: use a dedicated key, set a monthly spend cap at console.anthropic.com, and revoke it there any time. Your key is never included in backups.

Verify, don't trust: this app's full source code is public — github.com/gelithe/docs (the cloudflare-app folder). Anyone technical can confirm the key goes only to /api/chat → api.anthropic.com and is never stored.
